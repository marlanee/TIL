# 팀 프로젝트 리뷰
---
### 소감
3일간 팀 프로젝트를 진행했다. 명목상으로는 3일이지만, 실제로 작업한 시간은 공식적으로 10시간 남짓이다.  
2일차에 집에서 5시간 정도 개인 작업을 진행했으니, 나는 15시간 정도 진행한 작업이다.  

처음에는 막연했다. 한 번도 들어보지 못한 java, Vue 같은 용어들이 등장했고,  
프로젝트 명세서를 분석해서 목표를 잘게 쪼개 팀원별로 역할을 부여해야 했다.  
1일차는 AI에 명세서 분석을 던지고, 그 결과물로 주어진 엑셀 파일을 작성하고 끝났다.  
우리는 앞으로 어떤 일이 펼쳐질지도 모른 채 역할 분담을 하고 즐겁게 집으로 향했다.  

이틀차, 개발을 시작했다. 우리는 아는게 없어 AI에 의존할수밖에 없었다.
우리는 AI의 명령도 제대로 수행하지 못했다. 시작부터 문제였다. 어떤 형식의 파일을 만들어서 작업을 시작해야 하는지,  
작업을 한 후에 Github merge가 잘 될지, Ai가 작성해준 세부 계획에 대한 의심.  
이런 것들이 발목이 아닌 뒷목을 잡았더랬다. 그 한 발자국을 떼는 것이 참 힘들었다.  

2일차 작업은 촉박했다. SSAFY는 생각보다 빡빡했다.  
자유를 포기하고 AI에 전적으로 의존했다. 어떻게 시작해야 하는지 AI를 고문해 알아냈다.  
이런 저런 명령을 입력하니 처음 보는 파일들이 생성되었다. 웹 사이트 작업을 하도록 판을 깔아준 것이다.  
그때는 이 사실도 몰랐다. 공식적 2일차는 여기서 끝났다. 

집에 가서 야간 작업을 하기 시작했다. 조금 분했다. 무력한 기분이 싫었다.  
AI의 명령을 따라 만든 플랫폼의 토대에 카카오 맵과 챗 봇 기능을 추가했다.  
집에 가서 작업한 것은 저 두 가지 기능이었다. 한 줄로 적어놓으니 별 것 아닌 것 처럼 보인다.  
실제로 많은 개발자들에게는 별 것 아닌 작업일 것이다. 그러나 나에겐 매우 어렵게 느껴졌다.
API 생성부터 추가, AI의 기능 망치기 등 각종 시행착오들을 처음 겪었다.  

3일차 아침이 밝았다. 팀원에게 내가 만든 버전을 공유했다. 그래도 열심히 해서 그런지 다들 좋아했다. 
팀원에게 내가 잘 못하는 UX/UI 작업을 맡겼다. 결과는 훌륭했다.  
이후 내가 발표하고 제출해서 팀 프로젝트를 끝냈다.  

#### 부족한 점
- 협업을 의심하며 내가 모든 작업을 직접 하려고 했다
    - 팀원들이 맡은 작업은 그대로 맡겨두는 게 낫다.
- 프로젝트의 구조를 몰랐다.
    - 어떻게 플랫폼이 만들어지는지 몰랐다.
    - 그러니 어떻게 역할을 맡아 merge 하는지도 몰랐다
    - 그래서 머리 박치기 하며 내가 떠맡으려고 했다
- 결과물의 퀄리티가 아쉬웠다.
    - 부실했던 계획, 기능조차 제대로 구현하지 못했다
    - 다른 조와 비교하니, 수준 미달이었다.
- 프로젝트가 끝날 때까지, 오류를 해결하는 방법을 찾지 못했다.
    - 단편적인 오류들을 AI로 임기응변을 한 것이 전부였다.
    - 왜 과거의 기능이 망가지는지, 이 현상을 어떻게 해결하는지 몰랐다.
    - **다음에 다시 이런 프로젝트를 한다고 해도, 어떻게 프롬프트를 짜서 질문할지 모르겠다.**

#### 피드백
- 팀 프로젝트의 경우, 우선 맡은 역할과 임무에만 충실할 것.
    - 막힐 경우 팀원과 상황을 공유할 것
- 이번 프로젝트를 하며 협업의 구조를 조금은 이해했다.
    - Github로 협업을 어떻게 하는지 이해했다.
    - 그러나 혼자 떠맡은 느낌이 강하다
    - 답답하더라도 앞으론 Github를 활발하게 사용하자.
- 부실한 기획이 부실한 결과로 이어졌다.
    - 내가 이해하지 못하고 AI에 의존해 수립한 계획
    - 이해 없이 정확한 프롬프트와 방향을 제시할 수 없었다
    - Vscode의 copilot 활용법을 향상시켜야 한다.
- 코드 블럭화, diff 직접 눈으로 확인하기, 기존 기능 확인
    - 함수 및 변수 분리 코딩 요청하기
    - 바이브 코딩 이후에는 Diff 창 띄워서 변경사항 직접 눈으로 확인
        - 코딩에 익숙해지기 전까지는 Ai의 도움을 받자
    - 기존 기능이 잘 작동하는지 확인하기
