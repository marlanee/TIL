- dict (딕셔너리): key - value 쌍으로 이루어진, 순서와 중복이 없는 변경 가능한 자료형
  - 의미가 있으면서 순서가 필요 없는 데이터를 관리할 때 사용함
  - **순서가 없으니 index도 없음 / 시험 문제 낚시 주의!!**
  - Key를 통해 Value에 접근함 / 사전에서 단어의 뜻을 확인하는 것처럼 빠르게 접근함.
  - Key는 중복될 수 없음: 고유해야 함
  - Key로는 변경 불가능한 자료형만 사용 가능함
```python
# 딕셔너리 표현
my_dict_1 = {}
my_dict_2 = {'key': 'value'}
my_dict_3 = {'apple': 12, 'list': [1, 2, 3]}
print(my_dict_1)  # {}
print(my_dict_2)  # {'key': 'value'}
print(my_dict_3)  # {'apple': 12, 'list': [1, 2, 3]}

# 딕셔너리는 키에 접근해 값을 얻어냄
my_dict = {'name': '홍길동', 'age': 25}
print(my_dict['name'])  # '홍길동'
print(my_dict['test'])  # KeyError: 'test'

# 딕셔너리 값 추가 및 변경
my_dict = {'apple': 12, 'list': [1, 2, 3]}
# 추가
my_dict['banana'] = 50
print(my_dict)  # {'apple': 12, 'list': [1, 2, 3], 'banana': 50}

# 변경
my_dict['apple'] = 100
print(my_dict)  # {'apple': 100, 'list': [1, 2, 3], 'banana': 50}

```

- set (세트): 순서와 중복이 없는 변경 가능한 자료형
  - 집합의 관계 문제에 쓰는 함수임
```python
# 세트 표현
my_set_1 = set()
my_set_2 = {1, 2, 3}
my_set_3 = {1, 1, 1}
print(my_set_1)  # set()
print(my_set_2)  # {1, 2, 3}
print(my_set_3)  # {1}

# 세트의 집합 연산
my_set_1 = {1, 2, 3}
my_set_2 = {3, 6, 9}

# 합집합
print(my_set_1 | my_set_2)  # {1, 2, 3, 6, 9}

# 차집합
print(my_set_1 - my_set_2)  # {1, 2}

# 교집합
print(my_set_1 & my_set_2)  # {3}
```

- 불변과 가변(mutable vs immutable)
  - 불변: 변경 불가, 안전성, 예측 가능 / str, tuple, range
  - 가변: 변경 가능, 유연성, 효율성 / list, dict, set

- 형변환: 한 데이터 타입을 다른 데이터 타입으로 변환하는 과정
  - 암시적 형변환: 컴퓨터가 자동으로 처리하는 형변환
```python
# 암시적 형변환

# 불리언(bool)과 정수(int)의 덧셈
print(True + 3)  # 4
# 불리언간의 덧셈
print(True + False)  # 1
```
  - 명시적 형변환: 개발자가 직접 함수로 지정하여 변환

- 연산자: `+ - * / // % ** -`
- 복합 연산자: 연산과 할당이 함께 이루어짐
  - 가독성이 낮아서 비추천
  - 코드가 짧다고 좋은게 아님. 누구나 쉽게 읽을 수 있어야 함.
- 비교 연산자: 두 값을 비교, 옳고 그름을 True or False로 반환
  - `is`는 정체성을, `==`는 가치를 비교함
  - 대체로 `==`을 사용함
  - is 연산자는 Singleton 객체를 비교할 때 사용 / None, True, False
```python
# 왜 is 대신 ==를 사용해야 하나?
# 1(정수)과 True(불리언)는 다른 객체이다.
print(1 is True)  # False
# 1과 True의 '값'은 논리적으로 같다.
print(1 == True)  # True

# 2(정수)와 2.0(실수)은 다른 객체이다.
print(2 is 2.0)  # False
# 2와 2.0의 '값'은 논리적으로 같다.
print(2 == 2.0)  # True

# is 연산자는 언제 사용하는가?
# 싱글턴 객체 비교할 때
x = None
```

- 논리 연산자: 여러 개의 조건을 조합 또는 True/False 값을 뒤집을 때 사용
```python
# 논리 연산자 & 비교 연산자
num = 15
result = (num > 10) and (num % 2 == 0)
print(result)  # False

name = 'Alice'

result = (name == 'Alice') or (age == 30)
```

- 단축 평가: 논리 연산에서 두 번째 피연산자를 평가하지 않고 결과를 결정  
*공참타?*  
  - `And`, `or` 연산자에서 단축 평가를 사용함
```python
# 단축 평가

# -------------------------------------------------------------
# [1] 둘 다 값이 있는 경우
# -------------------------------------------------------------
# 준비물 1: 내용이 있는 문자열
item1 = '지도'
# 준비물 2: 내용이 있는 문자열
item2 = '나침반'
result = item1 and item2
print(f'최종적으로 챙긴 물건: {result}')
# >> 최종적으로 챙긴 물건: 나침반

# -------------------------------------------------------------
# [2] 뒤쪽이 거짓인 경우
# -------------------------------------------------------------
item1 = '지도'
# 준비물 2: 내용이 없는 빈 문자열
item2 = ''
result = item1 and item2
print(f'최종적으로 챙긴 물건: "{result}"')
# >> 최종적으로 챙긴 물건: ''


# -------------------------------------------------------------
# [3] 앞쪽이 거짓인 경우 => 뒤쪽은 아예 평가하지 않음
# -------------------------------------------------------------
# 준비물 1: 내용이 없는 빈 문자열
item1 = ''
item2 = '나침반'
result = item1 and item2
print(f'최종적으로 챙긴 물건: "{result}"')
# >> 최종적으로 챙긴 물건: ''
```

- 멤버십 연산자: 요소가 컬렉션 안에 포함되어있는지 확인
```python
# 멤버십 연산자

word = 'hello'
numbers = [1, 2, 3, 4, 5]

# -------------------------------------------------------------
# [1] in - 포함되어 있으면 True
# -------------------------------------------------------------
print('h' in word)  # True
print('z' in word)  # False

# -------------------------------------------------------------
# [2] not in - 포함되어 있지 않으면 True
# -------------------------------------------------------------
print(4 not in numbers)  # False
print(6 not in numbers)  # True
```
