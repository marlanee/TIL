### 함수 정의
  - def 키워드로 시작
  - def 키워드 이후 함수 이름 작성 (변수처럼 명료하게)
  - 함수는 필요한 경우 결과를 반환할 수 있음
    - return 키워드 이후 반환할 값을 명시
    - return 문은 결과를 반환하며 함수를 종료함
    - return 문이 없다면 None이 반환됨
  - 함수 호출
    - 함수를 사용하기 위해서 호출이 필요
  - 함수의 이름과 소괄호를 활용
```python
# 함수의 정의와 호출
def make_sum(pram1, pram2):   # 함수 정의
 """이것은 두 수를 받아
    두 수의 합을 반환하는 함수입니다.
    >>> make_sum(1, 2)
    3
    """
    result = pram1 + pram2
    return result

result = make_sum(100, 30)   # 함수 호출
print(result)  # 130

# print() 함수는 반환값이 없다.
value = print('hello world')
print(value)
"""hello world
None
"""

# return문이 아예 없는 함수도 마찬가지로 None을 반환
def my_func():
    print('hello')

result = my_func()
print(result)
```

### 함수의 인자 종류
- Positional Arguments (위치 인자)
  - 함수 호출 시 인자의 위치에 따라 전달되는 인자
```python
# 1. Positional Arguments (위치 인자)
def greet(name, age):
    print(f'안녕하세요, {name}님! {age}살이시군요.')
# 뭘 하든, 문자열을 쓸 때는 따옴표를 넣어야 한다.
greet('Alice', 25)  # 안녕하세요, Alice님! 25살이시군요.
greet(25, 'Alice')  # 안녕하세요, 25님! Alice살이시군요.
greet('Alice')  # TypeError: greet() missing 1 required positional argument: 'age'
```
- Default Argument Values (기본 인자 값)
  - 함수 정의에서 매개변수에 Default 값을 할당하는 것
```python
# 2. Default Argument Values
def greet(name, age=20):
    print(f'안녕하세요, {name}님! {age}살이시군요.')
#함수에 print 기능을 넣어두었으므로, 함수 호출만으로도 print 출력이 된다.
greet('Bob')  # 안녕하세요, Bob님! 20살이시군요.
greet('Charlie', 40)  # 안녕하세요, Charlie님! 40살이시군요.
```
- Keyword Arguments (키워드 인자)
  - 함수 호출 시 인자의 이름과 함께 값을 전달
  - 인자의 순서는 중요하지 않음. 인자의 이름만 명시하면 됨
  - **단, 호출 시 키워드 인자는 위치 인자 뒤에 위치해야 함** / 정의할때도 마찬가지다
    - 순서가 깨지면 위치 인자는 길을 잃게 된다
```python
# 3. Keyword Arguments
def greet(name, age):
    print(f'안녕하세요, {name}님! {age}살이시군요.')

greet(name='Dave', age=35)  # 안녕하세요, Dave님! 35살이시군요.
greet(age=35, name='Dave')  # 안녕하세요, Dave님! 35살이시군요.
greet(age=35, 'Dave')  # Positional argument cannot appear after keyword arguments
```
- Arbitary Argument Lists (임의의 인자 목록)
  - 정해지지 않은 개수의 인자를 처리하는 인자
  - 함수 정의 시 매개변수 앞에 '*'를 붙여 사용
  - 여러 개의 인자를 tuple로 처리
```python
# 4. Arbitrary Argument Lists
def calculate_sum(*args):   # 함수를 정의할 때, 마지막에 : 을 붙이는 것을 잊지 마라.
    print(args)  # (1, 100, 5000, 30)
    print(type(args))  # <class 'tuple'>
# 임의 인자 목록은 튜플을 만드는 함수인 듯 하다
calculate_sum(1, 100, 5000, 30)
```
- Arbitary Keyword Argument Lists (임의의 키워드 인자 목록)
  - 정해지지 않은 개수의 키워드 인자를 처리하는 인자
  - 함수 정의 시 매개변수 앞에 '**'를 붙여 사용
  - 여러 개의 인자를 dictionary로 묶어 처리
```python
# 5. Arbitrary Keyword Argument Lists
def print_info(**kwargs):
    print(kwargs)

print_info(name='Eve', age=30)  # {'name': 'Eve', 'age': 30}
# 임의 키워드 인자 목록은 딕셔너리를 만드는 함수인 듯 하다.
```

### 재귀 함수 (Recursion)
- 팩토리얼이 대표적인 재귀 함수 표현임
- 특정 알고리즘 식을 표현할 때 변수 사용이 줄어들고, 코드 가독성이 높아짐
- 1개 이상의 base case (종료되는 상황)이 존재하고, 수렴하도록 작성
- 반복되는 호출이 종료 조건을 향하도록 해야 함
```python
def factorial(n):
    # 종료 조건: n이 0이면 1을 반환
    if n == 1:
        return 1
    else:
        # 재귀 호출: n과 n-1의 팩토리얼을 곱한 결과를 반환
        return n * factorial(n - 1)   # factorial(4)를 구해야 하므로, 다시 연산 시작

# 팩토리얼 계산 예시
print(factorial(5))  # 120
```

### 내장 함수 (Built-in function)
- 파이썬에 기본적으로 내장된 함수
- 바퀴를 다시 발명하지 마라!
```python
numbers = [1, 2, 3, 4, 5]

print(numbers)  # [1, 2, 3, 4, 5]
print(len(numbers))  # 5
print(max(numbers))  # 5
print(min(numbers))  # 1
print(sum(numbers))  # 15
print(sorted(numbers, reverse=True))  # [5, 4, 3, 2, 1]
```

### 함수와 Scope
- 함수는 코드 내부에 local scope를 생성하며, 그 외의 global scope로 구분
  - 변수의 수명주기와 연관이 있다
```python
# Scope 예시
def func():
    num = 20
    print('local', num)  # local 20

func()

print('global', num)  # NameError: name 'num' is not defined
```
- 함수는 LEGB Rule에 따라 변수를 찾는다.
  - ex) sum을 변수로 설정하면, 더 이상 sum을 Built-in Function으로 사용하지 못한다.
```python
# 내장 함수 sum의 이름을 사용해버려서 오류가 발생하는 예시
print(sum)  # <built-in function sum>
print(sum(range(3)))  # 3
sum = 5
print(sum)  # 5
print(sum(range(3)))  # TypeError: 'int' object is not callable

# LEGB Rule 퀴즈
x = 'G'
y = 'G'

def outer_func():
    x = 'E'
    y = 'E'

    def inner_func(y):
        z = 'L'
        print(x, y, z)  # E P L

    inner_func('P')
    print(x, y)  # E E

outer_func()
print(x, y)  # G G
```
### global 키워드
- 변수의 scope를 전역 범위로 지정하기 위해 사용
- 함수 내에서 전역 변수를 수정할 때 사용
```python
num = 0  # 전역 변수

def increment():
    global num  # num를 전역 변수로 선언
    num += 1

print(num)  # 0
increment()
print(num)  # 1
```

### 단일 책임 원칙 (Single Responsibility Principle)
- 함수 설계 원칙
  - 명확한 목적
    - 함수는 한 가지 작업만 수행한다
    - 함수 이름으로 목적을 정확하게 표현
  - 책임 분리 - *블럭 쌓기처럼*
    - 기능을 분리한다. 검증, 처리, 저장 등의 기능을 별도 함수로 분리한다.
    - 각 함수는 독립적 동작이 가능해야 한다
  - 유지보수성
    - 작은 단위의 함수로 나누어, 코드 수정 시 영향 범위를 최소화한다.
```python
# 잘못된 설계 예시 (여러 책임이 섞인 함수)
def process_user_data(user_data):
    # 책임 1: 데이터 유효성 검사
    if len(user_data['password']) < 8:
        raise ValueError('비밀번호는 8자 이상이어야 합니다')

    # 책임 2: 비밀번호 암호화 및 저장
    user_data['password'] = hash_password(user_data['password'])
    db.users.insert(user_data)

    # 책임 3: 이메일 발송
    send_email(user_data['email'], '가입을 환영합니다!')

# 올바른 설계 예시 (책임을 분리한 함수들)   # 유저 데이터를 3개의 함수로 쪼갰다.
def validate_password(password):
    """비밀번호 유효성 검사"""
    if len(password) < 8:
        raise ValueError('비밀번호는 8자 이상이어야 합니다')

def save_user(user_data):
    """비밀번호 암호화 및 저장"""
    user_data['password'] = hash_password(user_data['password'])
    db.users.insert(user_data)

def send_welcome_email(email):
    """환영 이메일 발송"""
    send_email(email, '가입을 환영합니다!')
```

### Packing and Unpacking
- 패킹: 여러 개의 데이터를 하나의 컬렉션으로 모아 담는 과정
- 함수 정의 시 패킹 방식으로 동작한다
```python
# 1. 기본 패킹
packed_values = 1, 2, 3, 4, 5
print(packed_values)  # (1, 2, 3, 4, 5)

# 2. '*' 을 활용한 패킹 (함수 매개변수 작성 시)
def my_func(*args):
    print(args)  # (1, 2, 3, 4, 5)
    print(type(args))  # <class 'tuple'>
```
- 언패킹: 컬렉션에 담겨있는 데이터를 개별 요소로 펼쳐 놓는 과정
- 함수 호출 시 언패킹 방식으로 동작한다
```python
# 언패킹
a, b, c, d, e = packed_values
print(a, b, c, d, e)  # 1 2 3 4 5

# '*' 을 활용한 언패킹 (함수 인자 전달)
def my_function(x, y, z):
    print(x, y, z)

names = ['alice', 'jane', 'peter']
my_function(*names)  # alice jane peter
```

### 참고 파트
- 함수의 return, 반환의 원칙
  - **함수는 언제나 단 하나의 값(객체)만 반환할 수 있다**
  - 여러 값을 반환하는 경우에도 하나의 튜플로 패킹하여 반환한다.
```python
def get_user_info():
    name = 'Alice'
    age = 30
    # 콤마(,)로 여러 값을 반환하는 것처럼 보임
    return name, age


# 하지만 반환된 값을 user_data변수에 담아 확인해보면
user_data = get_user_info()

# 단 하나의 튜플을 반환하는 것 입니다.
print(user_data)  # ('Alice', 30)
```

- 람다 표현식 (Lambda expressions)
  - 익명 함수를 만드는 데 사용됨 - 1회용 함수라는 뜻이다
  - 한 줄로 간단한 함수를 정의. 특수한 상황에서만 사용
```python
# 람다 표현식 적용 전
def addition(x, y):
    return x + y

# 람다 표현식 적용 후
lambda x, y: x + y
```
