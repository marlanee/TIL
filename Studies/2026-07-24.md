### Dictionary
- 해시 테이블을 사용하여 키-값 쌍을 관리
### Dictionary Method
- `.get(key[,default])`: key의 value를 반환
  - `dic[key]`와 비슷함
  - 차이점: 키가 없으면 None 혹은 default를 반환
- `.keys()`: dictionary의 key를 반환 / 반복 가능
- `.values()`: dictionary의 value를 반환 / 반복 가능
- `.items()`: dictionary의 (key, value)를 반환 / 반복 가능
- `.setdefault(key[,default])`: key에 연결된 value를 반환
  - key가 없다면, dictionary에 key : default (value) 를 추가
- `.update([other])` : other dictionary를 기존 dictionary에 추가함. 중복될 시 덮어씀.
- `.pop(key[,default])`: key를 제거하고 value를 반환
  - key가 없으면 에러 또는 default를 반환
- `.clear()`: dictionary의 모든 key, value를 제거함
```python
# get
person = {'name': 'Alice', 'age': 25}
print(person.get('name'))
print(person.get('country'))
print(person.get('country', 'Unknown'))
# print(person['country'])  # KeyError: 'country'

# keys
person = {'name': 'Alice', 'age': 25}
print(person.keys())  # dict_keys(['name', 'age'])
for item in person.keys():
    print(item)

# values
person = {'name': 'Alice', 'age': 25}
print(person.values())  # dict_values(['Alice', 25])
for item in person.values():
    print(item)

# items
person = {'name': 'Alice', 'age': 25}
print(person.items())  # dict_items([('name', 'Alice'), ('age', 25)])
for key, value in person.items():
    print(key, value)

# setdefault
person = {'name': 'Alice', 'age': 25}
print(person.setdefault('country', 'KOREA'))  # KOREA
print(person)  # {'name': 'Alice', 'age': 25, 'country': 'KOREA'}

# update
person = {'name': 'Alice', 'age': 25}
other_person = {'name': 'Jane', 'country': 'KOREA'}

person.update(other_person)
print(person)  # {'name': 'Jane', 'age': 25, 'country': 'KOREA'}

person.update(age=100, address='SEOUL')
print(
    person
)  # {'name': 'Jane', 'age': 100, 'country': 'KOREA', 'address': 'SEOUL'}

# pop
person = {'name': 'Alice', 'age': 25}
print(person.pop('age'))  # 25
print(person)  # {'name': 'Alice'}
print(person.pop('country', None))  # None
# print(person.pop('country'))  # KeyError: 'country'

# clear
person = {'name': 'Alice', 'age': 25}
person.clear()
print(person)  # {}
```

### 딕셔너리의 확장: defaultdict
- 파이썬 내장 모듈 `collections`에서 제공하는 딕셔너리의 확장판
  - `from collections import defaultdict`
- 딕셔너리에 존재하지 않는 key를 조회할 때, 자동으로 기본값을 생성하는 특수 도구
- 선언: `defaultdict(자료형)`
- 주의사항: 조회만 해도 키가 생성됨
  - Ex) `print(my_dict['missing_key'])`를 하면 0이 출력되면서 키가 추가됨

```python
# 개선: defaultdict 활용
from collections import defaultdict

text = 'banana'
counts = defaultdict(int)  # 기본값을 정수(0)로 설정   # counts라는 빈 딕셔너리 껍데기가 생성됨

for char in text:
    # 키 확인 불필요! 없으면 0으로 자동 생성 후 +1
    counts[char] += 1

print(counts)  # defaultdict(<class 'int'>, {'b': 1, 'a': 3, 'n': 2})

# 활용 예제
# 색상별 과일 분류
fruits = [('red', 'apple'), ('yellow', 'banana'), ('red', 'cherry')]
fruit_by_color = defaultdict(list)   # fruit_by_color는 defaultdict라는 타입이다.

for color, fruit in fruits:
    # color 키가 없으면 빈 리스트 생성 -> append 바로 가능
    fruit_by_color[color].append(fruit)

print(
    fruit_by_color
)  # defaultdict(<class 'list'>, {'red': ['apple', 'cherry'], 'yellow': ['banana']})
print(
    dict(fruit_by_color)   # defaultdict type을 dict type으로 형변환
)  # {'red': ['apple', 'cherry'], 'yellow': ['banana']}
```

### 세트 (Set)
- 고유한 항목들의 정렬되지 않은 컬렉션 / 비시퀀스
  - set도 해시 테이블을 사용해서 데이터를 저장함 -> 고유성을 가짐 -> 순서 상관없이 빠르게 추가, 삭제 등 작업을 처리함
### 세트 메서드
- `.add(x)`: 세트에 x를 추가
- `.update(iterable)`: 세트에 다른 iterable 요소를 추가
- `.clear()`: 세트의 모든 항목을 제거함
- `.remove(x)`: 세트에서 x를 제거함.
  - x가 없을 경우 keyError 발생
  - 개발에서는 Error가 뜨는게 좋은거다! 원인 파악 가능
- `.pop()`: 세트에서 임의의 요소를 제거하고 반환함
  - `list.pop([i])`과 달리 ()만 사용 가능하다
- `.discard(x)`: 세트에서 x를 제거함.
  - remove와 같은 기능이지만, 에러가 안 뜸

```python
# add / 매번 순서가 바뀜. 인덱스가 없기 때문
my_set = {'a', 'b', 'c', 1, 2, 3}
my_set.add(4)
print(my_set)   # 출력: {'b', 2, 3, 1, 4, 'a', 'c'}

my_set.add(4)
print(my_set)   # 출력: {1, 2, 3, 'b', 'c', 4, 'a'}

# update
my_set = {'a', 'b', 'c', 1, 2, 3}
my_set.update([1, 4, 5])
print(my_set)  # {'c', 2, 3, 1, 'b', 4, 5, 'a'}

# clear
my_set = {'a', 'b', 'c', 1, 2, 3}
my_set.clear()
print(my_set)  # set()

# remove
my_set = {'a', 'b', 'c', 1, 2, 3}
my_set.remove(2)
print(my_set)
# my_set.remove(10)  # KeyError: 10

# pop
my_set = {'a', 'b', 'c', 1, 2, 3}
element = my_set.pop()
print(element)
print(my_set)

# discard
my_set = {'a', 'b', 'c', 1, 2, 3}
my_set.discard(2)
print(my_set)
my_set.discard(10)
```

### 해시 테이블 (hash table)
- key, value를 짝지어 저장하는 자료구조
- 원리: keys -> hash function -> buckets (해시 값)
  - buckets를 index로 삼아 데이터를 저장, 검색
- hash (해시): 임의의 크기를 가진 데이터를 고정된 크기의 고유한 값으로 변환하는 것
  - hash function (해시 함수)가 이 변환을 수행한다
    - 임의 크기 데이터 -> 고정 길이 (정수)로 변환하는 함수. 정수가 buckets (해시 값)이다.
- `.pop()` 메서드 사용시, 처음으로 발견되는 buckets의 key가 추출됨
  - 해시 난수화: 문자열은 매번 buckets 값이 바뀜. 즉, 임의임
    - 함수가 바뀌는게 아님. 난수 시드가 바뀌는 것임
    - 원리: hash function에 난수 시드가 매번 다르게 설정됨
- hashable: `hash()`로 해시 값을 구할 수 있는 객체
  - 일반적으로 불변 타입이어야 hashable임
  - 가변형 객체 (Ex) list, dict, set는 해시 불가능
  - 따라서, set, dict의 key에는 hashable 만 저장 가능
- dictionary는 삽입 순서가 보장되어 반복문 적용이 가능하다
  - 그러나, 내부 구현은 여전히 해시 테이블이다. 인덱스가 없다
