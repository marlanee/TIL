## Git Branch
What is branch?
### [Branch]
- Branch: 작업 공간을 분리, 독립 작업을 돕는 도구
- Branch의 장점
  - **독립된 개발 환경: 원본(master)가 안전함**
  - 매칭: 하나의 작업 - 하나의 브랜치: 체계적 개발 가능
  - 손쉬운 생성과 이동
- Master Branch
  - 기본 브랜치: 배포 가능한 상태로 유지
  - 기준점: 다른 브랜치의 시작 지점
  - 변경사항 통합: 최종 검증 후 브랜치를 master branch에 병합
- git command
- ```bash
  - git branch: 목록 확인
  - git branch -r: 원격 저장소 목록 확인
  - git branch <브랜치 이름>: 생성
  - git branch -d <브랜치 이름>: 병합된 브랜치 삭제
  - git branch -D <브랜치 이름>: 강제 삭제
  ```
- git switch
- ```
  - git switch <다른 브랜치 이름>: 다른 브랜치로 전환
  - git switch -c <브랜치 이름>: 새 브랜치 생성 후 전환
  - git switch -c <브랜치 이름> <commit ID>: 특정 커밋에서 새 브랜치 생성 후 전환
  ```

### [HEAD]
*HEAD는 마우스 포인트 커서와 비슷한 듯. 
가리키는 곳에 작업을 하는 점과 닮음*
- git switch 주의사항
  - **우선사항: Working Directory 버전 관리 확인**
    - commit하지 않은 파일이 남아있을 경우, 원본(master)에 영향을 끼치게 된다.
  - git의 버전 관리를 받지 않는 파일이기 때문임: Staging area에 올라가지 않아서 둥둥 떠 다님
  - git status를 수시로 사용, 작업 중인 파일 확인
- Branch 이동시 항상 최신 commmit으로 이동함
---
### Git Merge
Merge? 병합이란 뜻인 것 같다.
```
Git merge <병합 브랜치 이름>
```
- 주의사항
  - 수신 브랜치 확인
  - commit 상태 확인
- Merge 종류
  1. Fast-Forward Merge
     - 병합이 아님
     - 수신 브랜치를 병합 브랜치 상태로 만듦
     - 수신 브랜치를 병합 브랜치 단계까지 시킴
  2. 3-Way Merge
     -  *공통 조상이 필요하다고 함. 생물학인줄 알았음*
     -  *Fast-forward merge와 달리 작업 branch와 master branch의 길이 달라졌을 때 사용하는 듯*
     -  **새로운 commmit이 생긴다: Merge commmit**
---
### Merge Conflict
병합하려는 두 브랜치가 동일한 파일의 동일한 부분의 내용이 다를 경우 충돌 발생
- 3-Way merge에서만 Conflict가 발생한다.
