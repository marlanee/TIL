- Markdown: 글 구조 자동 변환 시스템.
  - 문법
    - 제목(# 개수로 h1 ~ h6까지 표현)
    - 목록: 1. -, * 등
    - 텍스트 강조: **강조**, *기울임*, ~~취소선~~
    - 구분선
    ---
    > 인용문
    인용문 내용
    
    - 링크: ["설명"]("사이트 주소")
    - 이미지: !["설명"]("이미지 주소")
      - 주소는 로컬 상대 주소 또는 온라인 주소
    - 코드 블록
      ```python 
      print("나는 고양이")
      ```
    - 인라인 코드: `print`
- README.md: 프로젝트나 코드의 소개서
  - 개발자의 기초적인 예의이자 소통 수단
  - 포트폴리오의 첫 인상을 결정하는 가장 중요한 파일

# Remote repository
- commit을 github에 올리는 것
- remote 명령어를 배워보자
  - git remote add "파일명" "Github 주소"
  - git push "파일명" master
  - git clone "Github 주소"
  - git pull "파일명" master
- 세부사항은 Notion에 적혀있음. 옮겨적기 귀찮아서 생략함
- .gitignore 파일: 안에 저장된 파일은 git의 추적을 받지 않음
  - gitignore.io를 참고할 것.
